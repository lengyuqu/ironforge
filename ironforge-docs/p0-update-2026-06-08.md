- **格式支持**: `.pom` (XML) → 自动提取；`.jar/.war/.aar/.ear` → 从文件名推断；`.tar.gz` → 搜索内部 .pom

- **命名约定**: Maven 包名存储为 `{groupId}:{artifactId}`
- **协议端点**: `GET /repos/{owner}/{name}/packages/maven/{group_id}/{artifact_id}/maven-metadata.xml`

## 剩余 P0: Docker/OCI 容器注册表

### 为什么 Docker/OCI 完全不同

| 维度 | 包注册表 (PyPI/Maven/Cargo) | Docker/OCI 注册表 |
|------|------|------|
| **协议** | 简单 HTTP REST（POST/GET 发布/下载）| OCI Distribution Spec v2（`/v2/` 前缀，7+ 个端点）|
| **存储** | 文件系统 blob（按 owner/repo/type/name/version 路径）| 内容寻址（SHA-256 摘要，自动去重）|
| **上传** | 单次全文件上传 | 分块上传 + 可恢复上传（PATCH/PUT）|
| **元数据** | 单一 manifest 文件 (Cargo.toml/PKG-INFO) | 多层次（manifest.json + config.json + layer blobs）|
| **客户端** | pip/pdm/mvn/gradle（使用注册表 API）| Docker CLI / containerd（使用原生 `/v2/` API）|
| **挂载** | 无 | 跨仓库 blob mount（节约存储）|
| **路由** | 与现有 REST API 共用 `/api/v1/repos/...` | 独立 `/v2/` 根路径，完全不同的路由树 |

### 需要的核心组件

#### 1. OCI Distribution API (HTTP 层)
```
GET  /v2/                                          — API 版本检查 (401 若需认证)
GET  /v2/{name}/tags/list                          — 列出 tags
HEAD /v2/{name}/manifests/{reference}              — 检查 manifest 是否存在
GET  /v2/{name}/manifests/{reference}              — 获取 manifest (JSON)
PUT  /v2/{name}/manifests/{reference}              — 上传 manifest
POST /v2/{name}/blobs/uploads/                     — 初始化 blob 上传
PATCH /v2/{name}/blobs/uploads/{uuid}              — 分块上传 (chunked)
PUT  /v2/{name}/blobs/uploads/{uuid}?digest={d}   — 完成上传
GET  /v2/{name}/blobs/{digest}                     — 下载 blob (layer)
POST /v2/{name}/blobs/uploads/?mount={d}&from={n}  — 跨仓库挂载
DELETE /v2/{name}/blobs/{digest}                   — 删除 blob
```

#### 2. 内容寻址存储层
- 按 `sha256:{digest}` 存储 blob，非文件名
- 自动去重：相同摘要只存一份
- 引用计数：manifest 被删除时检查 blob 是否仍被引用
- 基于现有 `PackageStorage` 扩展，或新建 `DockerStorage`

#### 3. OCI Manifest 处理
- 解析 `manifest.json`（v2 schema 2 或 OCI spec）
- 解析 image index（multi-arch）
- 解析 config.json（获取层列表 + 元数据）
- 与现有 `PackageAdapter` trait 集成

#### 4. 认证
- Docker 使用 Bearer token 通过 `WWW-Authenticate` header
- 需要 `/v2/token` 端点或复用现有 JWT 机制
