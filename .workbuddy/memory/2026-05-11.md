### 修复 toggle_star 500 错误（FTS5 触发器 bug）

**根因**：FTS5 迁移（m20260508_000005）中 `repos_fts_update` 等触发器使用了 FTS5 的 `'delete'` 特殊命令语法：
```sql
INSERT INTO repos_fts(repos_fts, rowid, name, description) VALUES('delete', old.id, old.name, ...);
```
FTS5 的 `'delete'` 命令不接受内容列值（只接受 rowid），导致 **任何 UPDATE repositories 语句**都触发 SQL logic error。

**修复**：
1. 新建迁移 `m20260511_000003_fix_fts5_triggers.rs`，用 `DELETE FROM repos_fts WHERE rowid = old.id` 替代 FTS5 `'delete'` 命令
2. 同时简化 `update_stars_count` / `update_forks_count`，去掉不再需要的 `updated_at` 更新
3. 修复测试 `test_star_repo`：去掉 `#[ignore]`，修正 stargazers 返回字段断言（`username` → `user_id`）
4. 结果：10/10 集成测试全绿

**踩坑**：FTS5 的 `'delete'`/`'insert'`/`'rebuild'` 命令是内部命令，语法限制严格。用标准 SQL `DELETE FROM` 更安全可靠。
