1. **reqwest Response.json() 消费 resp**：调用 `.json()` 后 `resp` 被 move，不能再用 `.headers()` — 必须先提取 header 再解析 body

6. **GitLab note.system 是 bool 不是 Option<bool>**：直接用 `if note.system` 而非 `.unwrap_or(false)`
7. **issue.body 和 pr.body 是 Option<String>**：DB 字段也是 Option，直接 `Set(issue.body.clone())`；但 issue_comment.body 是 String，需 `.unwrap_or_default()`

### 设计决策
- "先具体后通用"路由顺序：`cargo/index/` 和 `npm/` 的特定路由放在 `{pkg_type}/{pkg_name}` 通用路由之前
- 适配器工厂模式：`get_adapter()` 返回 `Box<dyn PackageAdapter>`，未识别的类型 fallback 到 `GenericAdapter`
- 发布流程：适配器自动提取 → 查询参数覆盖 → 最终验证 name/version 非空
- Sparse index 输出格式：每行一个 JSON 对象，符合 Cargo RFC 2789

- Maven 适配器需要 POM 解析 + checksum 生成
