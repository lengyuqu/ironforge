## IronForge p1-4 SSH git push — 全部完成 ✅

### 最终 BUG 根因
`send_response` 函数之前使用"两阶段"协议（先发 sideband flush，再发 plain pkt-lines）是错误的。
通过 `GIT_TRACE_PACKET=1` 对真实 `git-receive-pack` 抓包发现：

**正确的 sideband receive-pack 响应格式**：
1. 将整个 report-status 内容（`unpack ok\n` + `ok refs/...\n` + flush `0000`）打包成 pkt-lines 序列，写入内存缓冲区
2. 将该缓冲区整体作为 **sideband band-1 数据**发送（即一个大 pkt-line，前缀字节 `\x01`）
3. 最后发 sideband flush `0000` 结束整个响应

**实际线上抓到的字节**：
```
sideband< \1000eunpack ok0017ok refs/heads/main0000   ← band-1 含完整 report-status pkt-lines
sideband< 0000                                         ← sideband flush
```

### 修改文件
- `crates/rg-git/src/protocol/receive_pack.rs`：重写 `send_response`，改为先构建内存 report_buf，再整体 `write_sideband_data`，最后 `write_sideband_flush`
- `crates/rg-ssh/src/lib.rs`：保留 `stream.shutdown().await`（对 SSH 通道正常关闭有益，虽然本次 bug 原因不在于此）

### 端到端测试结果

- HTTP push/clone: 之前已验证通过，本次 HTTP `send_response` 代码路径相同，未回归
