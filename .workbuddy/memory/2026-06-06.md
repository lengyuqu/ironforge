## gix 迁移推进：Merge + Commit + Ref Delete 替换

### 替换内容
在 `crates/rg-core/src/pull_request/service.rs` 中，将以下 7 处 git CLI 调用替换为 gix API：

| 原 CLI | 替换函数 | 说明 |
|--------|---------|------|
| `git merge --no-ff` ×3 | `gix_merge_no_ff()` | 使用 `repo.merge_commits()` + `repo.commit()` 创建双父合并提交 |
| `git merge --squash` ×2 | `gix_squash_merge()` | 使用 `repo.merge_commits()` + `repo.commit()` 创建单父提交 |
| `git commit -m` ×2 | (合并到 gix_squash_merge) | squash 后 commit 由 gix commit API 执行 |
| `git update-ref -d` ×1 | `gix_delete_ref()` | 使用 `repo.edit_reference(Change::Delete)` |

### 新增辅助函数
- `gix_delete_ref()` — 删除引用
- `gix_merge_no_ff()` — no-ff 合并
- `gix_squash_merge()` — squash 合并
- `gix_merge_commits_to_tree()` — 核心合并逻辑，返回合并树 ID + 冲突

### 关键 API 使用
- `repo.merge_commits(our, their, labels, options)` → Outcome
- `outcome.tree_merge.tree.write()` → 写入合并树，获取 tree_id
- `repo.commit("HEAD", message, tree_id, [parent_ids])` → 创建提交
- `repo.tree_merge_options()` → 获取合并选项

## SSH Protocol V2 实现完成（20:18）

### 完成内容

#### 1. `crates/rg-git/src/pkt_line.rs`
- 修改 `read_pkt_line`, `read_pkt_lines_until_flush`, `read_text_line` 函数签名
- 从 `&mut BufReader<R>` 改为 `&mut R where R: AsyncRead + Unpin`
- 移除未使用的 `BufReader` 导入

#### 2. `crates/rg-git/src/protocol/v2.rs`
- 实现 `handle_v2_stream_impl` 函数（之前是空 stub）
- 使用 `tokio::io::split` 分离双向流为独立的读/写半部
- 实现命令处理循环：`ls-refs`, `fetch`, `object-info` 命令
- 正确使用 `BufReader::new(read_half)` 且只创建一次，在循环中复用

#### 3. `crates/rg-ssh/src/lib.rs`
- 在 `SshHandler` 结构体添加 `git_protocol_version: String` 字段（默认 "1"）
- 实现 `env_request` 方法捕获 `GIT_PROTOCOL=version=2` 环境变量
- 修改 `exec_request` 根据 `git_protocol_version` 路由到 V2 处理器

### 踩坑记录

1. **BufReader 移动后使用错误**：初始实现在循环中多次调用 `BufReader::new(read_half)`，导致 use-after-move 错误。修复：只在循环外创建一次 `let mut reader = BufReader::new(read_half)`，然后复用 `reader`。

2. **russh Handler trait 签名不匹配**：
   - `env_request` 必须有 5 个参数：`(channel_id, name, value, session)`
   - 返回类型必须是 `Result<(), Self::Error>`（不是 `Result<bool, ...>`）
